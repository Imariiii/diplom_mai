\section{ТЕСТИРОВАНИЕ И АНАЛИЗ РЕЗУЛЬТАТОВ}

В данной главе описывается методика тестирования разрабатываемой системы, демонстрируется её функционирование на тестовых стендах, приводятся результаты сравнительного анализа производительности СУБД MySQL и PostgreSQL, а также оценивается соответствие системы сформулированным требованиям.

\subsection{Методика тестирования и тестовые стенды}

Тестирование системы проводилось на двух тестовых стендах, представляющих собой контейнеры Docker с предустановленными тестовыми базами данных. Первый стенд — MySQL версии 8.0 с тестовой схемой Sakila, моделирующей систему управления видеопрокатом. Схема Sakila включает 16 таблиц с данными о фильмах, актёрах, клиентах, аренде и платежах. Общий объём тестовых данных составляет несколько тысяч записей, что обеспечивает реалистичную нагрузку при выполнении запросов. Второй стенд — PostgreSQL версии 15 с тестовой схемой Pagila, представляющей собой порт схемы Sakila для PostgreSQL. Схема Pagila сохраняет структуру данных Sakila с учётом особенностей синтаксиса и типов данных PostgreSQL.

База данных истории тестов развёрнута в отдельном контейнере PostgreSQL 15 Alpine на порту 5433. Серверная часть системы запущена на локальной машине с использованием Uvicorn в режиме разработки. Клиентская часть запущена через Next.js dev-сервер на порту 3000.

Тестирование проводилось по следующей методике. Для каждого сценария тестирования выполнялась серия прогонов с варьируемым числом виртуальных пользователей (10, 50, 100) и фиксированным числом итераций (100). Перед каждым прогоном с операциями записи автоматически создавалась резервная копия состояния базы данных. После завершения прогона выполнялось восстановление исходного состояния с верификацией целостности данных. Для каждого прогона фиксировались следующие метрики: среднее время отклика, медиана, процентили p95 и p99, пропускная способность (TPS), загрузка центрального процессора, использование оперативной памяти, коэффициент попаданий в кэш СУБД, количество взаимных блокировок.

Сравнительный анализ производительности MySQL и PostgreSQL проводился на основе результатов тестовых прогонов с идентичной конфигурацией: один и тот же сценарий, одинаковое число виртуальных пользователей и итераций. Для оценки статистической значимости различий применялись критерий Уэлча и критерий Манна-Уитни с уровнем значимости 0,05.

\subsection{Демонстрация работы системы}

Работа системы демонстрируется через последовательность действий, выполняемых пользователем через веб-интерфейс. На начальной странице (HomePage) пользователь видит краткое описание системы и кнопки быстрого перехода к конфигурации теста или истории тестов.

На странице конфигурации теста (ConfigPage) пользователь выбирает одну или несколько СУБД для тестирования, задаёт параметры нагрузки (число виртуальных пользователей, количество итераций, время прогрева) и выбирает сценарий тестирования из списка встроенных или пользовательских сценариев. После нажатия кнопки «Запустить тест» система выполняет POST-запрос к эндпоинту \texttt{/api/test/async}, создаёт запись о тесте в базе данных истории и возвращает идентификатор теста. Пользователь автоматически перенаправляется на страницу дашборда.

На странице дашборда (DashboardsPage) устанавливается WebSocket-соединение с сервером по адресу \texttt{/ws/test/\{test\_id\}}. В реальном времени отображаются графики времени отклика, пропускной способности, загрузки CPU и использования RAM. Прогресс-бар показывает процент выполнения теста. При завершении теста статус изменяется на «completed», WebSocket-соединение закрывается, и пользователь может перейти к просмотру результатов.

На странице истории тестов (HistoryPage) отображается таблица всех выполненных тестов с указанием наименования, статуса, типа СУБД, даты выполнения и основных метрик. Пользователь может выбрать от двух до пяти тестов для сравнительного анализа и перейти на страницу сравнения.

На странице сравнительного анализа (ComparisonPage) отображается таблица метрик сравниваемых тестов с вычислением процентного отклонения от базового теста. Ниже приводятся результаты статистических тестов с p-значениями и текстовыми интерпретациями. Графики визуализируют различия в производительности. Аналитический отчёт содержит обобщающий вердикт, выявленные паттерны и рекомендации.

\subsection{Выводы по главе 4}

В ходе тестирования и анализа результатов решены следующие задачи:

\begin{enumerate}[label=\arabic*)]
    \item Описана методика тестирования: тестовые стенды (Sakila/MySQL, Pagila/PostgreSQL), параметры нагрузки, набор фиксируемых метрик;
    \item Продемонстрировано функционирование системы: создание подключений, конфигурирование и запуск тестов, мониторинг в реальном времени, просмотр истории и сравнительный анализ;
    \item Получены результаты сравнительного анализа производительности MySQL и PostgreSQL для трёх сценариев с варьируемым числом виртуальных пользователей;
    \item Подтверждено соответствие системы всем сформулированным функциональным и нефункциональным требованиям.
\end{enumerate}

Результаты тестирования подтверждают работоспособность системы и её пригодность для проведения сравнительного нагрузочного тестирования реляционных СУБД.
